# 🚀 立即开始 - 获取 TaskFlow Windows 可执行文件

## 问题
您无法在本地安装 Visual Studio C++ 工具，因此无法直接编译 Windows 应用。

## 解决方案
使用 **GitHub Actions** 自动编译您的应用，无需本地 Visual Studio！

---

## 📋 完整步骤（预计 10 分钟）

### 第一步：创建 GitHub 仓库（2 分钟）

1. **访问**: https://github.com/new
2. **填写信息**：
   - Repository name: `TaskFlow`
   - Description: "任务管理工具" (可选)
   - **Visibility: Public** (必须选公开，GitHub Actions 免费)
   - **⚠️ 重要**: 不要勾选 "Add a README file"
3. **创建仓库**：点击 "Create repository"
4. **复制 URL**：如 `https://github.com/你的用户名/TaskFlow.git`

---

### 第二步：推送代码到 GitHub（3 分钟）

打开 **PowerShell (管理员)** 并执行：

```powershell
# 进入项目目录
cd C:\taskflow

# 初始化 Git
git init

# 添加所有文件
git add .

# 提交代码
git commit -m "Initial commit: TaskFlow"

# 重命名分支
git branch -M main

# ⚠️ 替换为你的仓库 URL
git remote add origin https://github.com/你的用户名/TaskFlow.git

# 推送到 GitHub
git push -u origin main
```

**或者使用自动化脚本**（推荐）：
```powershell
cd C:\taskflow
.\AUTOMATED_SETUP.ps1
```

---

### 第三步：等待自动构建（5-10 分钟）

1. **访问你的仓库**：打开上述 GitHub 仓库 URL
2. **点击 "Actions" 标签页**
3. **找到 "Build Windows Release" 工作流**
4. **等待状态变为绿色勾号 ✅**
   - 🟡 黄色圆圈 = 运行中
   - 🟢 绿色勾号 = 成功
   - 🔴 红色 X = 失败

---

### 第四步：下载可执行文件（2 分钟）

1. **点击工作流运行**（绿色勾号）
2. **滚动到页面底部 "Artifacts" 部分**
3. **下载 `taskflow-windows.zip`**
4. **解压文件**，找到：
   - `TaskFlow.exe` (主程序)
   - `flutter_windows.dll` (Flutter 引擎)
   - 其他 DLL 文件

---

## 📦 构建产物说明

| 文件 | 说明 |
|------|------|
| `TaskFlow.exe` | 主程序，双击运行 |
| `flutter_windows.dll` | Flutter 运行时 |

**文件大小**: 约 30-50 MB（压缩后）

## ✅ 验证安装

1. 将解压后的文件复制到任意 Windows 电脑
2. 双击 `TaskFlow.exe` 运行
3. 应用应该正常启动（无需安装）

## 🛠️ 故障排除

### 工作流失败

**查看错误日志**：
1. 访问 Actions 标签页
2. 点击失败的工作流
3. 展开失败的步骤查看详细错误

**常见错误及解决**：

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `intl` 版本冲突 | 依赖不兼容 | 已在 workflow 中自动修复 |
| 找不到 `cl.exe` | Visual Studio 未安装 | GitHub Actions 自动安装 |
| 网络超时 | 下载依赖超时 | 重试工作流 |

### 找不到下载链接

- 确保工作流已完成（绿色勾号）
- 只有成功的构建才有下载链接
- 如果工作流仍在运行，请等待完成

### 推送代码失败

**检查以下事项**：
1. 仓库 URL 是否正确
2. 是否有推送权限
3. 网络连接是否正常
4. Git 是否正确配置（用户名、邮箱）

**手动推送命令**：
```powershell
cd C:\taskflow
git remote -v  # 查看远程仓库
git push -u origin main --force  # 强制推送
```

## 🔄 更新应用

4. 完成后从 Actions 下载新版本
